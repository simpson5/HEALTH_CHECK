# 운동 기록 웹앱 기획서

> 작성일: 2026-03-18
> 목적: 플릭 AI 운동 앱의 핵심 기능을 웹으로 구현하여 별도 앱 없이 운동 기록

---

## 1. 현재 문제

- 플릭 AI 앱에서 운동 기록 → 데이터를 건강관리 시스템으로 **수동 전달** 필요
- 앱이 별도라서 이중 관리
- 기록 귀찮음

## 2. 목표

> 플릭 AI의 핵심 기능을 **health.simpson-space.com/workout.html**에 구현.
> 운동 기록이 `simpson_data.json`에 **직접 저장**되어 대시보드에 즉시 반영.

---

## 3. 플릭 AI 앱 분석 (스크린샷 기반)

### 루틴 화면
- 운동 목록 리스트 (순서대로)
- 각 운동: 이름 + 타겟 부위 + 일러스트
- 하단: [시작] [운동 추가]

### 운동 실행 화면
- 운동 일러스트 (동작 이미지)
- 운동명 + 볼륨 표시
- 세트 테이블: **세트 | KG | 회 | 완료** 체크
- W = 워밍업 세트
- 휴식 타이머 (90초 기본)
- 세트 추가/삭제

### Simpson 현재 루틴 (6종)
1. 머신 체스트 프레스 — 가슴, 어깨
2. 랫 풀 다운 — 등, 이두
3. 머신 숄더 프레스 — 어깨, 삼두
4. 레그 익스텐션 — 대퇴사두
5. 라잉 레그 컬 (라잉 햄스트링 컬) — 햄스트링, 종아리
6. 머신 시티드 로우 — 등, 이두

## 6. 핵심 기능

### 필수 (MVP)
| 기능 | 설명 |
|------|------|
| 루틴 선택 | 풀루틴/상체/하체 선택 → 운동 목록 로드 |
| 세트 기록 | KG, 횟수 입력 + 완료 체크 |
| 휴식 타이머 | 세트 완료 시 자동 카운트다운 (90초 기본) |
| 운동 간 이동 | 이전/다음 버튼 |
| 자동 저장 | 운동 완료 시 JSON 저장 + git commit |
| 볼륨 계산 | 세트 × KG × 회 자동 합산 |
| 이전 기록 참조 | 지난번 무게/횟수 자동 불러오기 |

### 추후 (v2)
| 기능 | 설명 |
|------|------|
| 운동 일러스트 | 각 운동 동작 이미지 |
| 세트 추가/삭제 | 동적으로 세트 관리 |
| 루틴 커스텀 | 운동 추가/제거/순서 변경 |
| 볼륨 그래프 | 주간/월간 볼륨 변화 차트 |
| 1RM 추정 | 무게×횟수 기반 1RM 계산 |
| 진동 알림 | 휴식 끝나면 폰 진동 |

## 7. 기술 구현

### 프론트엔드
- `static/workout-session.html` — 운동 실행 페이지 (별도 HTML)
- 순수 HTML/CSS/JS (프레임워크 없음)
- localStorage로 진행 중 세션 임시 저장 (브라우저 꺼져도 복구)

### 백엔드
- `POST /api/exercise` — 운동 기록 저장 API 추가
- server.py에 엔드포인트 1개 추가

### 데이터
- `simpson_data.json`에 `workout_routines` 추가
- `exercise_records` 구조 확장 (세트 상세 포함)

## 8. 입력 방식 비교

| | 플릭 AI 앱 | 웹 구현 |
|--|-----------|---------|
| 운동 선택 | 앱 내 | 루틴 탭에서 선택 |
| KG/회 입력 | 앱 내 | 웹 폼 (숫자 키패드) |
| 완료 체크 | 탭 | 탭 |
| 타이머 | 앱 내 | 웹 타이머 |
| 데이터 전달 | **수동 (CSV/스크린샷)** | **자동 (JSON 직접 저장)** |
| 대시보드 반영 | 불가 | **즉시 반영** |
