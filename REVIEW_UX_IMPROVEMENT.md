# 사용성 개선 검토서

> 작성일: 2026-03-18
> 목적: 식단 기록 프로세스 최소화 방안 검토

---

## 1. 현재 흐름

```
1. 폰에서 📷 버튼 탭 → 사진 촬영 → 서버 업로드
2. 클로드한테 "점심 김치찌개 반인분" 전달
3. 클로드가 JSON 업데이트 + 사진 연결
4. 폰 새로고침 → 반영 확인
```
**소요 시간**: ~2분
**불편 포인트**: 2번 단계에서 별도로 클로드한테 말해야 함

---

## 2. 개선 방안

### 방안 A: 사진 + 메모 동시 전송 (즉시 적용 가능)

```
[📷 사진찍기]
[김치찌개 반인분        ] ← 메모 한 줄
[전송]
```

- **구현**: 대시보드 📷 버튼에 메모 입력칸 추가
- **효과**: 사진 + 메모가 서버에 같이 저장됨
- **처리**: 클로드가 photos/ + 메모 파일 확인 → JSON 자동 업데이트
- **난이도**: 낮음 (프론트 수정만)
- **비용**: 없음
- **단점**: 여전히 클로드가 확인 후 수동 처리 필요

### 방안 B: 음성 입력 (단기 개선)

```
[📷 사진찍기]
[🎤] "김치찌개 반인분" → 텍스트 자동 변환
[전송]
```

- **구현**: Web Speech API (브라우저 내장, 무료)
- **효과**: 타이핑 불필요, 말하면 됨
- **난이도**: 중간 (JS 추가)
- **비용**: 없음
- **단점**: 조용한 환경에서만 정확, iOS Safari 지원 제한적

### 방안 C: 사진만 올리면 자동 분석 (궁극적 목표)

```
[📷 사진찍기] → 끝
서버가 자동으로: 음식 인식 → 영양소 추정 → JSON 업데이트
```

- **구현**: 서버에서 Claude API 호출 (vision)
- **효과**: 사진 한 장이 끝. 메모도 필요없음
- **난이도**: 중간
- **비용**: Claude API 호출당 ~$0.01~0.03 (하루 3~5회 = 월 $1~5)
- **정확도**: 90%+ (패키지 제품은 거의 정확, 일반 음식은 추정)
- **단점**: API 키 필요, 네트워크 필수, 추정 오차 존재

### 방안 D: 텔레그램 봇 (대안 채널)

```
텔레그램 앱 → 사진 + "점심 김치찌개" 전송 → 봇이 처리
```

- **구현**: Python telegram bot + Claude API
- **효과**: 별도 웹 접속 불필요, 카톡처럼 자연스러운 UX
- **난이도**: 높음
- **비용**: Claude API 비용 + 서버 상시 구동
- **단점**: 별도 앱 필요, 구현 복잡

### 방안 E: Apple 단축어 (iOS 위젯)

```
홈 화면 위젯 1탭 → 카메라 → 음성 메모 → 자동 전송
```

- **구현**: iOS Shortcuts + HTTP 요청
- **효과**: 앱 전환 없이 홈 화면에서 바로
- **난이도**: 중간
- **비용**: 없음
- **단점**: iOS 전용, 설정 복잡

## 3. 추천 로드맵

```
지금          →      1주 내       →       2주 내
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
방안 A             방안 B            방안 C
사진+메모 전송     음성 입력 추가    AI 자동 분석
                                    (사진만 찍으면 끝)
```

### Step 1: 방안 A (즉시)
- 📷 버튼 클릭 → 사진 + 메모 칸 팝업
- 서버에 사진 + 메모 텍스트 같이 저장
- 클로드가 주기적으로 확인 → JSON 업데이트

### Step 2: 방안 B (추가)
- 메모 칸 옆에 🎤 버튼
- 음성 → 텍스트 자동 변환
- "김치찌개 반인분" 말하면 텍스트로 들어감

### Step 3: 방안 C (최종)
- 사진 업로드 시 서버에서 Claude API vision 호출
- 음식 자동 인식 + 영양소 추정 + JSON 자동 업데이트
- **사용자 행동: 📷 한 번 = 끝**
