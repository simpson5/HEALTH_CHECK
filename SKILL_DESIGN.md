# /health 스킬 설계서

> 작성일: 2026-03-18
> 목적: Simpson 건강관리 통합 스킬 — 하나의 명령으로 모든 건강 데이터 처리

---

## 1. 스킬 개요

| 항목 | 내용 |
|------|------|
| 스킬명 | `/health` |
| 역할 | 입력 자동 판별 → 분석 → 코멘트 → JSON 기록 → git commit |
| 트리거 | `/health` 또는 건강 관련 대화 감지 시 자동 |

---

## 2. 자동 판별 로직

사용자 입력을 보고 **자동으로 유형 판별**:

```
입력                           → 판별 유형
─────────────────────────────────────────
인바디 사진 첨부               → inbody
"점심 김치찌개 먹었어"          → diet
식단 사진 첨부                 → diet
"오늘 운동 ~했어"              → exercise
"마운자로 5mg으로 올렸어"       → medication
"이번 주 어땠어?"              → weekly_analysis
"현황 보여줘"                  → status
```

---

## 3. 유형별 처리 플로우

### 3-1. 식단 (diet)

**입력 형태**:
- 텍스트: "점심 김치찌개 반인분 먹었어"
- 사진: 📷 업로드 (photos/ 폴더 확인)
- 사진 + 텍스트: "아침에 이거 먹었어" + 사진

**처리 순서**:
1. 사진 있으면 → photos/ 확인 → 음식 파악
2. 영양소 추정 (칼로리, 단백질, 탄수, 지방)
3. `simpson_data.json` → `diet_records` 추가
4. 사진 파일명 정규화: `photos/YYYY-MM-DD_끼니_음식명.jpg`

**코멘트 예시**:
```
✅ 점심 기록 완료: 김치찌개 + 공기밥 (0.5인분)
   칼로리 325kcal | 단백질 10g | 탄수 44g | 지방 9.5g

⚠️ 오늘 단백질 현황: 10g / 110g (100g 부족)
   👉 저녁에 닭가슴살 200g(단백질 46g) 추천
```

### 3-2. 인바디 (inbody)

**입력 형태**:
- 인바디 결과지 사진
- 직접 수치 전달: "체중 107, 골격근 32, 체지방 51..."

**처리 순서**:
1. 사진이면 → 수치 읽기 (OCR/vision)
2. 전 측정 대비 변화량 자동 계산
3. `simpson_data.json` → `inbody_records` 추가
4. 사진 → `photos/YYYY-MM-DD_인바디.jpg`

**코멘트 예시**:
```
📊 인바디 분석 (D+16, 2026-03-25)

   체중: 107.2kg (▼1.5kg)   ✅
   골격근: 32.5kg (▲0.2kg)  ✅ 근육 유지!
   체지방: 50.1kg (▼1.7kg)  ✅ 지방 감소!
   체지방률: 46.7% (▼1.0%)  ✅
   BMI: 36.7 (▼0.5)

   💪 이번 주는 지방 위주로 빠졌다! 단백질 섭취 효과 보이는 중.
   📌 계속 단백질 110g 유지하자.
```

### 3-3. 운동 (exercise)

**입력 형태**:
- "오늘 트레드밀 30분 + 체스트프레스 3세트 했어"
- "오늘 유산소만 40분"
- 운동 기록 사진 (앱 스크린샷 등)

**처리 순서**:
1. 운동 종류/시간/세트/횟수 파악
2. 칼로리 소모 추정
3. `simpson_data.json` → `exercise_records` 추가

**코멘트 예시**:
```
💪 운동 기록 완료 (2026-03-18)

   🏃 유산소: 경사 트레드밀 30분 (~250kcal)
   🏋️ 근력:
     - 체스트 프레스: 3세트 × 12회 × 30kg
     - 랫 풀다운: 3세트 × 10회 × 25kg

   총 소모: ~400kcal
   ✅ 근력 2종 완료. 하체도 빠지지 말자!
```

**데이터 구조**:
```json
{
  "date": "2026-03-18",
  "exercises": [
    {
      "type": "cardio",
      "name": "경사 트레드밀",
      "duration_min": 30,
      "calories_burned": 250,
      "memo": "경사 12%, 속도 5.5"
    },
    {
      "type": "strength",
      "name": "체스트 프레스",
      "sets": 3,
      "reps": 12,
      "weight_kg": 30,
      "memo": ""
    }
  ],
  "total_duration_min": 50,
  "total_calories_burned": 400,
  "memo": ""
}
```

### 3-4. 투약 (medication)

**입력 형태**:
- "마운자로 5mg으로 올렸어"
- "이번 주부터 7.5mg"
- "부작용으로 메스꺼움 있어"

**처리 순서**:
1. 용량 변경/부작용 파악
2. `simpson_data.json` → `medication_records` 추가

**코멘트 예시**:
```
💊 투약 기록 업데이트
   2.5mg → 5mg (2026-03-24, D+15)

   📌 용량 올리면 첫 1~2주 식욕 억제 더 강해질 수 있음
   👉 단백질 쉐이크로 최소 섭취량 확보하자
```

### 3-5. 주간 분석 (weekly_analysis)

**트리거**: "이번 주 어땠어?", "주간 분석", "리포트"

**처리 순서**:
1. 해당 주 인바디/식단/운동 데이터 종합
2. 평균 단백질, 칼로리, 운동 횟수 계산
3. 체중/근육/지방 변화 분석
4. 개선 포인트 + 추천 도출
5. `simpson_data.json` → `weekly_analysis` 추가

### 3-6. 현황 (status)

**트리거**: "현황", "지금 상태", "요약"

**처리**: JSON 읽고 현재 상태 요약 출력 (기록 없음, 조회만)

## 4. 공통 마무리 (모든 유형)

모든 처리 후 반드시:

```
1. simpson_data.json 저장
2. git add simpson_data.json photos/
3. git commit -m "health: {유형} - {요약}"
   예: "health: diet - 0318 점심 김치찌개"
   예: "health: inbody - D+9 108.7kg"
   예: "health: exercise - 유산소30분+근력2종"
```

## 6. 입력 데이터 전달 방식 검토

| 데이터 유형 | 전달 방식 | 비고 |
|------------|----------|------|
| 식단 텍스트 | 자연어 | "점심 김치찌개 반인분" |
| 식단 사진 | 📷 버튼 → photos/ | 클로드가 확인 |
| 인바디 | 사진 첨부 or 수치 텍스트 | 사진이면 vision 분석 |
| 운동 | 자연어 | "트레드밀 30분 체프 3세트" |
| 투약 | 자연어 | "5mg으로 올렸어" |

### 운동 입력 간소화 방안
- 매일 루틴이 비슷하므로 → "오늘 풀루틴" 하면 기본 세트 자동 입력
- 변경사항만 말하면 됨: "오늘 랫풀다운 빼고 나머지 다 했어"
- 무게/횟수 변경: "체프 35kg으로 올렸어"
