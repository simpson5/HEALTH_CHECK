## 1. 현재 구조

### 네비게이션

```
상단 탭 (7개): 홈 | 식단 | 체중 | 운동 | 달력 | 기록 | 🤖AI
하단 네비 (3개): 📊대시보드 | 📖가이드 | ⚙️설정
별도 페이지: 운동세션 | 음식관리
```

### 페이지 (11개)

| 페이지 | 접근 방법 | 역할 |
|--------|----------|------|
| Home | 상단탭 '홈' | 오늘 요약 (체중/미션/단백질/식단/운동) |
| Diet | 상단탭 '식단' | 날짜별 식단 조회 |
| Weight | 상단탭 '체중' | 체중 그래프 + 인바디 |
| Exercise | 상단탭 '운동' | 주간현황 + 운동시작 + 가이드 |
| Calendar | 상단탭 '달력' | 월간/주간 달력 + 미션 달성 |
| History | 상단탭 '기록' | 일일/주간 리포트 + 투약 + 약물농도 |
| AI | 상단탭 'AI' | 사진/텍스트 → AI 분석 + 빠른 등록 |
| WorkoutSession | 운동탭 → 운동시작 | 운동 기록 (세트/회/KG) |
| Guide | 하단네비 '가이드' | 식단/운동 가이드 |
| Foods | (숨겨짐) | 음식 데이터 관리 |
| Settings | 하단네비 '설정' | 토큰 설정 + 서버 상태 |

## 2. 문제점

### 2-1. 네비게이션 혼란

| 문제 | 상세 |
|------|------|
| **상단 7탭 과다** | 폰 화면에서 7개 탭이 한 줄에 안 들어감. 스크롤 필요 |
| **이중 네비** | 상단 탭 7개 + 하단 네비 3개 = 사용자가 어디를 봐야 할지 모름 |
| **하단 네비 역할 불명확** | 대시보드(=홈 탭과 동일), 가이드, 설정 — 왜 여기 있는지 이유 불분명 |
| **AI 탭이 이질적** | 다른 탭은 조회인데 AI만 입력+처리. 성격이 다름 |
| **음식관리 접근 불가** | 하단 네비에서 제거됨. 직접 URL 입력해야 접근 |

### 2-2. 데이터 입력 동선 불명확

| 작업 | 현재 동선 | 문제 |
|------|----------|------|
| 체중 기록 | AI 탭 → 텍스트 입력 or 나한테 말하기 | **직접 입력 폼 없음** |
| 식단 기록 | AI 탭 → 사진+텍스트 → 초안 → 확인 → 저장 | 단계 많음 |
| 빠른 식단 등록 | AI 탭 → 빠른 등록 버튼 | 괜찮지만 AI 탭 안에 숨어있음 |
| 운동 기록 | 운동 탭 → 운동 시작 → 별도 페이지 | 나쁘지 않음 |
| 인바디 등록 | CSV 업로드 → API | **웹 UI에서 직접 할 수 없음** |
| 투약 기록 | 나한테 말하기만 | **웹 UI 없음** |

### 2-3. 홈 화면 정보 과다

```
현재 홈 (위→아래):
  체중 카드
  목표 프로그레스 바
  오늘 미션 체크리스트
  단백질 프로그레스 바
  오늘 식단 요약
  최근 운동
  = 6개 섹션. 스크롤 길어짐.
```

| 문제 | 상세 |
|------|------|
| 정보가 많아서 핵심이 묻힘 | 한눈에 "오늘 잘 하고 있나?"를 못 봄 |
| 식단 요약이 중복 | 식단 탭과 동일한 정보 |
| 최근 운동 카드가 부실 | 운동 탭의 축소판일 뿐 |

### 2-4. 식단 탭 한계

| 문제 | 상세 |
|------|------|
| **조회만 가능** | 식단 추가/수정/삭제 불가 |
| 날짜 이동은 있지만 | 기록이 없는 날은 빈 화면 |
| 사진이 안 보이는 경우 많음 | 사진 없이 등록된 기록이 대부분 |

### 2-5. 체중 탭

| 문제 | 상세 |
|------|------|
| **체중 입력 폼 없음** | 그래프만 보이고 직접 입력 못 함 |
| 인바디 날짜 버튼이 작음 | pill 형태라 탭하기 어려움 |
| 체성분 차트 4개 탭 전환 | 유용하지만 차트마다 의미 이해 어려움 |

### 2-6. 운동 탭

| 문제 | 상세 |
|------|------|
| 서브탭 2단 (머신→부위) | 탭이 많아서 복잡 |
| 운동 시작 버튼이 글로우 펄스 | 눈에는 띄지만 과함 |
| 운동 가이드가 길어짐 | 28종 등록돼있어서 |

### 2-7. 달력 탭

| 문제 | 상세 |
|------|------|
| 월간 보기에서 칸이 작음 | 이모지는 보이지만 정보 부족 |
| 주간 보기가 세로로 길음 | 7일 × 130px = 스크롤 |
| 주간 요약이 기본적임 | 숫자만 나열 |

### 2-8. 기록 탭

| 문제 | 상세 |
|------|------|
| 서브탭 4개 (일일/주간/투약/약물) | 직관적이긴 하지만 데이터 없으면 빈 화면 |
| 약물 농도 차트 실용성 | 보기엔 좋지만 실제로 자주 보지 않음 |

### 2-9. AI 탭

| 문제 | 상세 |
|------|------|
| 처리 시간 10~30초 | 기다리는 동안 할 수 있는 게 없음 |
| 초안 확인 UI가 단순 | 수정 기능 없이 저장/취소만 |
| 빠른 등록 버튼이 가로 스크롤 | 6개 넘으면 스크롤 |
| 상담 결과가 카드 하나 | 대화 히스토리 없음 |

### 2-10. 디자인 전반

| 문제 | 상세 |
|------|------|
| **색상 너무 어두움** | 전체적으로 검정+어두운 회색, 콘텐츠 구분 어려움 |
| **카드 계층 불명확** | glass vs glass-bright 차이가 미미 |
| **여백 불규칙** | 페이지마다 패딩이 다름 |
| **폰트 크기 위계 약함** | 큰 숫자와 본문의 차이가 더 커야 함 |
| **애니메이션 부재** | 페이지 전환, 카드 등장 효과 없음 |
| **탭 전환 시 깜빡임** | 상태 리셋되면서 로딩 중 표시 |
| **카메라/파일 버튼 위치** | 오른쪽 하단 플로팅 — 다른 콘텐츠와 겹침 |

### 2-11. 모바일 사용성

| 문제 | 상세 |
|------|------|
| 터치 타겟 작음 | 일부 버튼/탭이 44px 미만 |
| 스크롤 방향 혼재 | 대부분 세로지만, 탭바/빠른등록은 가로 |
| 키보드 올라올 때 | 입력 필드가 가려질 수 있음 |
| 새로고침 의존 | 데이터 변경 후 자동 갱신 안 되는 경우 있음 |
