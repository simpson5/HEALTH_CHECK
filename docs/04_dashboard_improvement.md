# 대시보드 개선사항 정리

> 작성일: 2026-03-18
> 목적: 현재 대시보드 문제점 + 개선 방향 정리

---

## 1. 현재 탭 구조 및 문제점

| 탭 | 현재 내용 | 문제 |
|----|----------|------|
| 개요 | 목표 달성률 + 최신 지표 6개 + 경고 | 정보 과다, 핵심이 묻힘 |
| 그래프 | 체중/근육vs지방/체지방률/BMI/기초대사량 | 데이터 2건뿐, 아직 의미없음 |
| 식단 | **오늘 식단만** | 전날/과거 식단 못 봄 |
| 기록 | 인바디 타임라인 | OK |
| 예측 | 목표 달성 예측 | 하드코딩, 데이터 부족 |

### 빠져있는 것
- 매일 몸무게 기록 / 조회
- 운동 기록 조회
- 투약 이력 조회
- 과거 날짜 식단 조회

---

## 2. 새로운 데이터: 매일 몸무게

### 배경
- 인바디: 주 1회 (헬스장 방문)
- 몸무게: **매일 아침** 체중계 사진 → 클로드가 기록

### JSON 구조 추가
```json
"weight_records": [
  {
    "date": "2026-03-18",
    "weight_kg": 108.7,
    "photo": "photos/2026-03-18_체중.jpg",
    "memo": ""
  }
]
```

### 입력 방식
- 아침에 체중계 사진 📷 업로드
- 클로드가 사진 확인 → 숫자 읽기 → JSON 기록
- `/health` 스킬에 weight 유형 추가

## 3. 개선된 탭 구조 (안)

### 탭 구성

| 탭 | 내용 | 변경사항 |
|----|------|---------|
| **홈** | 오늘 한눈에 보기 | 기존 개요 간소화 |
| **식단** | 날짜별 식단 + 사진 | 날짜 이동 추가 |
| **체중** | 몸무게 그래프 + 인바디 | 신규 (핵심 탭) |
| **운동** | 운동 기록 + 가이드 | 신규 |
| **기록** | 투약 이력 + 주간 분석 | 기존 기록탭 확장 |

### 삭제/통합
- 예측 탭 → 홈에 한 줄로 통합 ("목표까지 28.7kg, 예상 2027년 중반")
- 그래프 탭 → 체중 탭에 통합
- 기존 기록(인바디 타임라인) → 체중 탭에 통합

## 4. 탭별 상세 설계

### 4-1. 홈 탭

```
┌─────────────────────────┐
│ 오늘 몸무게  108.7kg     │ ← 오늘 weight_record
│ ▼0.3kg (어제 대비)       │
│                         │
│ ━━━━━━━━━━━▓░░░ 13.3%   │ ← 목표 프로그레스 바
│ 113.1 → 80kg (28.7kg 남음)│
│                         │
│ ── 오늘 식단 ──          │
│ 점심: 김치찌개 325kcal    │
│ 단백질: 10g / 110g ⚠️    │
│                         │
│ ── 최근 운동 ──          │
│ 어제: 트레드밀 30분+근력3종│
│                         │
│ 💊 마운자로 2.5mg D+9    │
└─────────────────────────┘
```

핵심: **스크롤 없이 한 화면에 오늘 상태 전부 보기**

### 4-2. 식단 탭

## 6. /health 스킬 변경사항

### 새 유형: weight (매일 몸무게)

```
입력: 체중계 사진 업로드
판별: 사진에 숫자만 있으면 weight
처리:
  1. 사진에서 몸무게 읽기
  2. weight_records에 추가
  3. 어제 대비 변화 코멘트
  4. git commit
```

### 기존 유형 판별 보완

```
체중계 사진 (숫자만)     → weight (매일)
인바디 결과지 사진 (표)   → inbody (주간)
음식 사진               → diet
```
