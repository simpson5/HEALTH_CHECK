## 3. 식단 빠른 등록 (frequent_foods 원탭 등록)

### 컨셉
```
식단 탭 또는 홈 → [빠른 등록] 버튼
→ frequent_foods 목록 팝업
→ "닥터유 쉐이크+두유" 탭
→ 끼니 선택 (아침/점심/저녁/간식/보충제)
→ 바로 오늘 식단에 추가 (API 호출)
→ 대시보드 갱신
```

### 화면
```
┌─────────────────────────┐
│  빠른 등록               │
│                         │
│ [닥터유 쉐이크+두유]      │ ← 탭하면 바로 등록
│  262kcal · P36g          │
│                         │
│ [닥터유 파우더만]          │
│  172kcal · P24g          │
│                         │
│ [매일두유 고단백]          │
│  90kcal · P12g           │
│                         │
│ 끼니: ○보충제 ○간식 ○음료 │
│                         │
│ [등록] [취소]             │
└─────────────────────────┘
```

### 장점
- 클로드한테 말 안 해도 자주 먹는 음식은 **폰에서 직접 1탭 등록**
- 새 음식은 여전히 클로드가 처리

### 필요 API
- `POST /api/diet` — 식단 직접 추가 엔드포인트

## 3-1. 사이트에서 Claude Code 앱 실행 가능한가?

### 결론: 직접 실행은 불가능

웹사이트에서 폰의 Claude Code 앱을 직접 열 수 있는 공식 URL scheme이 없습니다.

### 대안
| 방법 | 가능성 |
|------|--------|
| `claude://` URL scheme | 공식 미지원, 불가 |
| 웹사이트에 "클로드에게 말하기" 버튼 | 버튼 누르면 클립보드에 텍스트 복사 → 앱 전환해서 붙여넣기 |
| PWA + 공유 기능 | Share API로 Claude 앱에 텍스트 공유 시도 (불안정) |
| **빠른 등록으로 대체** | frequent_foods는 사이트에서 직접 등록, 새 음식만 클로드 |

→ **빠른 등록 기능으로 클로드 호출 빈도를 줄이는 게 현실적**

```
│ ⚠️ 104g 부족              │
└─────────────────────────┘
```

### 구현
- renderHome()에 프로그레스 바 HTML 추가
- 색상: 30% 미만 빨강, 30~70% 주황, 70% 이상 초록

## 5. 일일/주간 리포트

### 일일 리포트
- **트리거**: "오늘 리포트" 또는 `/health 오늘 어땠어?`
- **내용**:
  - 오늘 식단 합계 (칼로리, 단백질, 탄수, 지방)
  - 단백질 목표 달성 여부
  - 오늘 운동 여부 + 요약
  - 체중 변화 (어제 대비)
  - 종합 평가 + 내일 조언

### 주간 리포트
- **트리거**: "이번 주 리포트" 또는 `/health 주간 분석`
- **내용**:
  - 체중 변화 (주 시작 vs 주 끝)
  - 평균 단백질/칼로리
  - 운동 횟수
  - 인바디 변화 (있으면)
  - 주간 분석 + 다음 주 추천

### 자동 vs 수동
- **현재**: 요청 시 생성 (수동)
- **추후**: 매일 저녁 10시 / 매주 월요일 자동 생성 (cron 또는 launchd)
- 자동 생성된 리포트는 JSON `daily_reports` / `weekly_analysis`에 저장
- 대시보드 기록 탭에서 조회 가능

## 6. 운동 유형별 입력 UI 개선

### 문제 1: 맨몸 운동에 KG 칸 불필요

| 운동 유형 | 입력 항목 |
|----------|----------|
| 머신 근력 | 세트 + **KG** + 회 + 완료 |
| 맨몸 (푸쉬업/스쿼트) | 세트 + 회 + 완료 (**KG 숨김**) |
| 케틀벨 | 세트 + **KG** + 회 + 완료 |
| 유산소 (트레드밀) | 시간 + 경사 + 속도 |

### 구현
- exercise_library에 `input_type` 필드 추가:
  - `"weight_reps"` — KG + 회 (머신, 케틀벨)
  - `"reps_only"` — 회만 (맨몸)
  - `"cardio"` — 시간/경사/속도
- workout-session.html에서 input_type에 따라 UI 분기

### 문제 2: 운동 종류 늘어나면 목록이 너무 길어짐

현재 9종 + 유산소 1종 = 10개. 앞으로 더 늘어날 수 있음.

### 해결: 카테고리 접기/펼치기

```
── 머신 근력 (6종) ──         ▾
  체스트 프레스 / 랫 풀다운 / ...

── 맨몸 (3종) ──              ▾
  케틀벨 스윙 / 벽 푸쉬업 / 맨몸 스쿼트

── 유산소 (1종) ──            ▾
  경사 트레드밀
```

### 구현
- exercise_library에 `group` 필드 추가:
  - `"machine"` — 머신 근력
  - `"bodyweight"` — 맨몸/프리웨이트
  - `"cardio"` — 유산소
- 운동 선택 화면 + 운동 탭 가이드 모두 그룹별 표시
- 각 그룹 접기/펼치기 가능
