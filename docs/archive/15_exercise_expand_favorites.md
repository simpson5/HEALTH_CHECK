## 3. 즐겨찾기 기능

### 문제
운동이 20~30종이 되면 매번 스크롤해서 찾아야 함.

### 해결: 즐겨찾기 (⭐)

```
운동 선택 화면:

⭐ 즐겨찾기 (5종)
  체스트 프레스
  랫 풀다운
  숄더 프레스
  레그 익스텐션
  레그 컬

── 상체 밀기 ──
  체스트 프레스 ⭐
  숄더 프레스 ⭐
  펙덱 플라이
  ...

── 상체 당기기 ──
  랫 풀다운 ⭐
  ...
```

### 데이터

DB에 `is_favorite` 필드 추가:

```sql
ALTER TABLE exercise_library ADD COLUMN is_favorite BOOLEAN DEFAULT 0;
```

### UI

- 운동 카드에 ⭐ 토글 버튼
- 즐겨찾기된 운동은 **상단에 별도 섹션**으로 표시
- 운동 세션에서도 즐겨찾기 우선 표시

### 토글 방식

운동 가이드에서 ⭐ 탭 → 즐겨찾기 등록/해제.
API: `PUT /api/exercise-library/:id/favorite`

## 4. 운동 추가 방식

### 사전 등록 vs 사용자 추가

| 방식 | 설명 |
|------|------|
| **사전 등록** | 위 목록을 DB에 미리 넣어둠. 헬스장에 없는 건 안 보이게 |
| **사용자 추가** | 웹에서 직접 운동 추가 가능 |
| **추천: 둘 다** | 사전 등록 + 없는 건 직접 추가 |

### 사용자 추가 UI (추후)

```
운동 탭 하단:
[+ 운동 추가]
  이름: [플랭크]
  부위: [코어]
  유형: ○근력(KG+회) ○맨몸(회만) ○유산소(시간)
  [저장]
```

Simpson이 다음에 헬스장에서 확인해줄 것:
- 펙덱/버터플라이 있는지

- 크런치 머신 있는지

확인되면 해당 운동만 등록합니다.
