# 기록 탭 재설계 계획서

> 작성일: 2026-03-24
> 문제: 기록 탭이 데이터를 전부 렌더링하는 구조. 1년치 쌓이면 사용 불가.

---

## 1. 현재 문제점

| 문제 | 상세 |
|------|------|
| 일일 리포트 전체 렌더링 | 데이터 늘어나면 스크롤 끝없이 길어짐 |
| 주간 분석 전체 렌더링 | 동일 |
| 투약 이력 전체 렌더링 | 동일 |
| UI 단조로움 | 회색 카드만 나열, 탐색 어려움 |
| 날짜 탐색 불가 | 특정 날짜/주간 리포트 찾기 어려움 |

---

## 2. 재설계 방향

### 핵심: 주간 단위로 탐색

식단 탭처럼 **◀▶ 주간 이동** 방식. 선택한 주의 데이터만 표시.

```
┌─────────────────────────┐
│ ◀  3/18 ~ 3/24 (이번 주) ▶ │ ← 주간 네비게이션
│                           │
│ [일일] [주간분석] [투약] [약물] │ ← 서브 탭
│                           │
│ ── 일일 리포트 ──           │
│ (해당 주 리포트만 표시)       │
│                           │
│  3/24 (화) A등급  ▾        │ ← 접기/펼치기
│  3/23 (월) A등급  ▾        │
│                           │
└─────────────────────────┘
```

---

## 3. 서브 탭 구성

| 탭 | 내용 | 렌더링 |
|----|------|--------|
| **일일** | 해당 주 일일 리포트 | 최대 7건 |
| **주간분석** | 해당 주 주간 분석 | 1건 |
| **투약** | 해당 주 투약 기록 | 0~1건 |
| **약물** | 마운자로 체내 농도 차트 | 1건 (차트) |

### 각 탭 상세

#### 일일 탭
- 해당 주(월~일)의 일일 리포트만 표시
- 각 리포트는 **접기/펼치기** — 기본 접힌 상태로 날짜+점수만 보임
- 펼치면 상세 (분석, 하이라이트, 식단 요약 등)

```
3/24 (화)  A  🔥1167kcal  P121g     ▾
3/23 (월)  D  🔥225kcal   P6g       ▾
  ↓ (탭하면 펼침)
3/23 (월)  D  🔥225kcal   P6g       ▴
  분석: 하루 총 칼로리 225kcal...
  ✅ ...
  ⚠️ ...
  💡 내일 조언: ...
```

#### 주간분석 탭
- 해당 주의 주간 분석 1건
- 없으면 "이 주 분석 없음"

#### 투약 탭
- 해당 주 투약 기록
- 다음 투약 예정일 표시

#### 약물 탭
- 마운자로 체내 농도 차트 (기존)
- 이건 주간 필터 없이 전체 표시 (차트 특성상)

## 4. UI 디자인

### 주간 네비게이션
```css
배경: var(--card)
테두리: var(--border)
좌우 화살표: var(--accent) 원형 버튼
날짜 텍스트: 굵게, 가운데 정렬
"이번 주" 배지: accent 색상 작은 태그
```

### 서브 탭
```css
배경: var(--card2)
활성: accent 배경 + 테두리
둥근 모서리 (pill 형태)
```

### 일일 리포트 카드 (접힌 상태)
```css
배경: var(--card)
좌측: 날짜 + 요일
가운데: 칼로리 + 단백질 칩
우측: 점수 (색상별) + ▾ 아이콘
한 줄로 컴팩트하게
```

### 일일 리포트 카드 (펼친 상태)
```css
접힌 상태 + 아래에:
  분석 텍스트
  하이라이트 목록
  식단 요약
  내일 조언
  구분선으로 섹션 분리
```

## 5. 데이터 처리

### 주간 필터링 로직
```
1. 현재 선택된 주의 월요일/일요일 계산
2. 일일 리포트: date >= 월요일 && date <= 일요일
3. 주간 분석: week_start가 해당 주에 포함
4. 투약 기록: date >= 월요일 && date <= 일요일
5. 렌더링: 필터된 데이터만
```

### 성능
- 최대 7건 (일일) + 1건 (주간) + 1건 (투약) = 최대 9건 렌더링
- 1년치 데이터여도 항상 9건 이하
